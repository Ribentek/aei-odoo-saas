### Admin Odoo — namespace `odoo-admin`

| Resource | Kind | Key Specs |
|:---|:---|:---|
| `odoo-admin` | Deployment | `odoo:18`, 1 replica, Recreate |
| `odoo-admin-svc` | Service | Ports 8069 + 8072 |
| `odoo-admin-data` | PVC | `local-path`, RWO, 20Gi |
| `odoo-admin-conf` | ConfigMap | `odoo.conf.tmpl` with `dbfilter=^admin$`, rendered via `render-config` init container |
| `odoo-admin-ingress` | Ingress | `admin.aeisoftware.com → odoo-admin-svc:8069` |

The admin pod uses **two init containers**:

1. **`copy-addon`** (`alpine/git`) — Clones SaaS addons, `subscription_oca`, `payment_qr_mercantil`, and MUK modules from GitHub
2. **`render-config`** (`alpine:3.19`) — Renders `odoo.conf.tmpl` replacing `REPLACE_DB_PASSWORD` and `REPLACE_ADMIN_PASSWD` with actual Secret values

```yaml
initContainers:
  - name: copy-addon
    image: alpine/git
    command:
      - sh
      - -c
      - |
        git clone --depth=1 -b "$ADDON_GIT_BRANCH" https://github.com/Ribentek/aei-odoo-saas.git /tmp/repo
        cp -r /tmp/repo/odoo_k8s_saas /mnt/extra-addons/
        cp -r /tmp/repo/odoo_k8s_saas_subscription /mnt/extra-addons/
        cp -r /tmp/repo/payment_qr_mercantil /mnt/extra-addons/
        git clone --depth=1 -b 18.0 https://github.com/Ribentek/odoo18-oca-contract.git /tmp/oca-contract
        cp -r /tmp/oca-contract/subscription_oca /mnt/extra-addons/
        git clone --depth=1 -b 19.0 https://github.com/muk-it/muk_base.git /tmp/muk-base
        for d in /tmp/muk-base/muk_*/; do cp -r "$d" /mnt/extra-addons/; done
  - name: render-config
    image: alpine:3.19
    command:
      - sh
      - -c
      - |
        sed -e "s|REPLACE_DB_PASSWORD|${DB_PASSWORD}|g" \
            -e "s|REPLACE_ADMIN_PASSWD|${ADMIN_PASSWD}|g" \
            /tmpl/odoo.conf.tmpl > /etc/odoo/odoo.conf
```

The addon volume (`odoo-addons`) is `emptyDir` — all addons are re-cloned on every pod restart. The branch is controlled by the ConfigMap key `addon-git-branch` (default: `main`).

### Per-Tenant Resources — namespace `odoo-<tenant_id>`

Applied by the portal in this order:

| # | Resource | Kind | Notes |
|:---|:---|:---|:---|
| 1 | `odoo-<tenant_id>` | Namespace | Labels: `managed-by=saas-portal`, `tenant=<id>` |
| 2 | `odoo-data` | PVC | `local-path`, RWO, `storage_gi` GB (default 10) |
| 3 | `odoo-secret` | Secret | Generated 32-char `DB_PASSWORD` + `ADMIN_PASSWD` |
| 4 | `odoo-conf` | ConfigMap | `odoo.conf` (see below) |
| 5 | `odoo` | Deployment | `odoo:18`, Recreate, resource limits below |
| 6 | `odoo` | Service | ClusterIP, 8069 + 8072 |
| 7 | `odoo-ingress` | Ingress | `<id>.aeisoftware.com → odoo:8069/8072` |

409 conflicts (already-exists) are silently ignored — provisioning is idempotent.

## odoo.conf (per tenant)

Generated by `k8s_utils/manifests.py::configmap_manifest()`:

```ini
[options]
db_host = postgres.aeisoftware.svc.cluster.local
db_port = 5432
db_user = odoo
db_password = %(DB_PASSWORD)s     ← injected from env via K8s Secret
admin_passwd = %(ADMIN_PASSWD)s   ← injected from env via K8s Secret
dbfilter = ^<tenant_id>$          ← hard-coded per tenant (no odoo_ prefix)
list_db = False
addons_path = /usr/lib/python3/dist-packages/odoo/addons
data_dir = /var/lib/odoo
workers = 2
max_cron_threads = 1
gevent_port = 8072
proxy_mode = True
```

## Ingress Rules

### portal-ingress (namespace `aeisoftware`)

```
traefik.ingress.kubernetes.io/router.entrypoints: web
host: portal.aeisoftware.com
path: /  → portal:8000
```

### odoo-admin-ingress (namespace `odoo-admin`)

```
host: admin.aeisoftware.com
/websocket → odoo-admin-svc:8072
/          → odoo-admin-svc:8069
```

### Per-tenant odoo-ingress (namespace `odoo-<id>`)

```
traefik.ingress.kubernetes.io/router.middlewares: kube-system-odoo-headers@kubernetescrd
host: <tenant_id>.aeisoftware.com
/websocket → odoo:8072
/          → odoo:8069
```
